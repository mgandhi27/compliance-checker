 Technical Standards Compliance Checker (A Hybrid BERT-Powered Framework)
 <br>
 Project Overview

The **Technical Standards Compliance Checker** is an AI-based, hybrid NLP framework designed to automate the verification of complex engineering designs against stringent international regulatory standards.

Traditional compliance verification is a grueling manual process; evaluating just eight clauses can consume hours of a highly skilled engineer's time. A major hurdle in automation has been the "vocabulary mismatch" between formal normative standard language (e.g., "differential output voltage") and informal engineering design prose.

This project solves that challenge by utilizing **BERT (Bidirectional Encoder Representations from Transformers)**. By leveraging BERT's deep contextual representations, the system translates both regulatory standards and design documents into dense mathematical vectors, allowing it to understand semantic meaning and intent rather than relying on brittle keyword matching.

 Key Features & Technical Details

* **BERT-Driven Semantic Understanding:** Utilizes transformer-based semantic embeddings to resolve the vocabulary mismatch, mapping informal engineering prose to formal normative requirements.
* **Hybrid Decision Engine:** Merges AI semantic reasoning (for complex process clauses) with deterministic regular expressions (for absolute numeric limits). The scoring equation heavily weights the rule-based engine (60%) for numeric specifications to prevent AI hallucinations and ensure out-of-spec values result in a hard failure.
* **Advanced NER Pipeline:** Deploys a custom spaCy Named Entity Recognition pipeline to decompose standard clauses into actionable data triplets (Subject, Obligation, Value), enabling structural linguistic understanding.
* **Numeric Normalization:** Integrates regular expressions paired with the `pint` unit registry library to automatically normalize disparate engineering units (e.g., converting "300 microns" and "0.3mm" to a common base for mathematical comparison).
* **High-Speed Vector Search:** Utilizes FAISS (Facebook AI Similarity Search) to index 384-dimensional dense vectors generated by `all-MiniLM-L6-v2`, enabling sub-linear time retrieval across massive document databases.
* **Industrial Reliability:** Incorporates defensive design patterns, including a 400-token sliding window with a 50-token overlap to bypass BERT's 512-token limit without losing context, and defensive JSON parsing to strip markdown fences.

Supported Technical Standards

The engine currently curates and evaluates 42 strict requirement clauses across six major industrial domains:

1. **IEEE 802.3-2022** (Electronics/Ethernet)
2. **IPC-2221B** (PCB Design)
3. **ISO 9001:2015** (Quality Management)
4. **NEC 2023** (Electrical Safety)
5. **BIS IS 732** (Electrical Installation)
6. **ASME Y14.5** (Mechanical GD&T)

 System Architecture

The core of the system operates on a custom **Six-Stage Compliance Analysis Pipeline**:

1. **Parse (Ingestion):** Extracts text from heterogeneous documents (PDF, DOCX) using `pdfplumber` and `pytesseract` OCR (300-600 DPI).
2. **Tokenize:** Splits text into subword units, preserving highly technical engineering vocabulary.
3. **Embed:** Converts tokens into dense semantic vectors, mapping meaning to geometric space.
4. **Retrieve:** Executes FAISS approximate nearest-neighbor search to identify design paragraphs aligning with standard clauses.
5. **Validate:** Passes extracted triplets through the Hybrid AI + Rule scoring equation.
6. **Report:** Aggregates findings into a traceable GUI matrix and exports downloadable CSV/PDF reports.

 Performance Metrics

The system has been rigorously stress-tested and calibrated.

* **Accuracy:** Achieves an overall weighted F1-score of 0.920, outperforming previous industry benchmarks (0.89 F1) in automated compliance.
* **Speed:** Completes a comprehensive 8-clause scan in just 2.4 seconds, with UI rendering latency remaining under 40ms even when processing up to 500 requirement clauses simultaneously.
* **Calibration Variance:** Maintains less than 1% error against historical industrial pass/fail rates (calibrated at 55% Pass, 23% Warn, 22% Fail).

 Future Roadmap

* **Server-Side API Upgrades:** Shifting heavy document parsing (PDF/DOCX) to a dedicated FastAPI Python backend using asynchronous Celery workers and Redis.
* **Multi-Standard Parallel Scanning:** Enabling a single design document to be evaluated against all six standard frameworks simultaneously to produce a comprehensive compliance heatmap.
* **PostgreSQL Persistence:** Implementing a robust database tier to maintain immutable audit trails and track version-to-version compliance regression across design iterations.
* **Dynamic Import Facility:** Developing a module for users to upload JSON-based clause registries to support internal company guidelines and updated regulatory editions.
